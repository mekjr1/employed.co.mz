Visual shift summary.
  • Buttons: 4 px radius + inset shadow → 8 px radius + flat, 3 variants (primary / outline / ghost), focus ring uses indigo 4 px halo instead of inset red glow.
  • Inputs: top-aligned labels → floating labels; red focus glow → indigo halo; .form-group → .form-floating.
  • Cards: .panel (grey heading bar, 4 px radius) → .card (transparent header, 12 px radius, soft drop-shadow).
  • Alerts: beige alert-warning → amber-tinted with leading glyph for accessibility.
  • Modals: BS5 modals are accessibility-clean by default (correct focus trap, ESC handling, ARIA) — replaces the peppelg:bootstrap-3-modal shim that client/lib/dialog.js wraps today.
  • Navbar: dropdowns become Popper-based; navbar-default → navbar navbar-expand-lg; mobile toggle uses navbar-toggler-icon (SVG) instead of three icon-bar spans.
  • Grid: .col-md-6 stays; the named .col-sm-* stack reflows slightly because BS5 breakpoints shifted (576 / 768 / 992 / 1200 / 1400). Some layouts may need a single column override.
  • JS coupling: BS5 drops jQuery requirement → can drop $('.modal').modal('show') jQuery calls in jobForms.js, job.js, header.js, dialog.js. Replace with new bootstrap.Modal(el).show().
  • Glyphicons: removed in BS4+. The icon-bar, glyphicon-chevron-* references in custom.bootstrap.less become dead code and are deleted.
  • LESS → SASS: BS5 ships SASS only. The two custom.bootstrap.*.less files get rebuilt as SASS variable overrides ($primary: #4F46E5; $border-radius: 8px; etc.). The Less-based journal.theme.import.less can be deleted.

Approval — what happens when you say go

If you greenlight this preview, the rebrand pass executes in this order so each PR is independently shippable:

  1. Step 0 — pick a logo concept from [design/logos/concepts.html](design/logos/concepts.html). The recommendation is Concept 01 (Geometric E with spark).
  2. Step 1 — logo production assets: refine the chosen concept; export logo.svg, logo-dark.svg, favicon.svg; regenerate the 5 PNG variants; flip manifest.json theme_color to #4F46E5. (closes surfaces 1–4 manifest portion + brand foundation)
  3. Step 2 — Tier 1 badges: Featured seal + Verified seal + status-pill LESS tokenisation. (closes surfaces 1, 2, 3)
  4. Step 3 — Tier 2 brand moments: spinner + 404 + no-jobs + designed OG card. (closes surfaces 5–8)
  5. Step 4 — Tier 3 system polish: email header partial + avatar SVG fallback. (closes surfaces 9, 11)
  6. Step 5 — Bootstrap 5 + custom icon set, paired together because both touch every template. Replaces FA4 with the custom SVG set (surface 10), rewrites the LESS overrides as SASS, swaps panel → card / label → badge / floating-label forms, drops jQuery modal calls, regenerates the visual regression baseline. (closes surface 10 + framework upgrade)
  7. Step 6 — full test battery on fresh Docker stack (Meteor unit + smoke chromium + journeys chromium + journeys firefox) to confirm no UAT regression.

Reply go + your logo-concept pick to start at Step 1. Reply with concept-level objections to refine this preview first.
