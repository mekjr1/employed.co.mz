# Employed — Brand kit

> A single source of truth for the Employed visual identity. Use this folder when you need a logo for a tweet, a t‑shirt, a header image, an email signature, or anything else off‑product.

The shipping app pulls a small slice of these assets into `public/images/`. Everything else here is reference material for marketing, merch, partner co‑branding, and social media.

## Regenerating raster assets

All PNG fallbacks under `public/images/` are checked in *and* regenerated by a single command:

```bash
npm run regen-brand-pngs
```

That runs [scripts/regen-brand-pngs.js](../scripts/regen-brand-pngs.js), which uses [`@resvg/resvg-js`](https://github.com/yisibl/resvg-js) (a pure-WASM SVG renderer with no native deps) to rasterise the SVGs in `brand/logo/`, `brand/social/og-card.svg` and `public/images/avatar.svg` into the exact sizes the app + manifest reference. Re-run it any time you edit a source SVG; the script only rewrites files whose bytes actually changed, so git diffs stay honest.

If for some reason you'd rather not use Node, the same outputs can be produced with `rsvg-convert` or ImageMagick:

```bash
# rsvg-convert (recommended — librsvg, sharpest output)
rsvg-convert -w 16  -h 16  brand/logo/favicon.svg     -o public/images/favicon-16x16.png
rsvg-convert -w 32  -h 32  brand/logo/favicon.svg     -o public/images/favicon-32x32.png
rsvg-convert -w 152 -h 152 brand/logo/logo-mark.svg   -o public/images/favicon.png
rsvg-convert -w 180 -h 180 brand/logo/logo-mark.svg   -o public/images/apple-touch-icon.png
rsvg-convert -w 192 -h 192 brand/logo/logo-mark.svg   -o public/images/icon-192x192.png
rsvg-convert -w 512 -h 512 brand/logo/logo-mark.svg   -o public/images/icon-512x512.png
rsvg-convert -w 192 -h 192 brand/logo/logo-mark.svg   -o public/images/maskable-icon-192x192.png
rsvg-convert -w 512 -h 512 brand/logo/logo-mark.svg   -o public/images/maskable-icon-512x512.png
rsvg-convert -w 1200 -h 630 brand/social/og-card.svg  -o public/images/og-card.png
rsvg-convert -w 200 -h 200 public/images/avatar.svg   -o public/images/avatar.png

# Or ImageMagick (alternative — slightly softer aa, more universally installed)
magick -background none -density 600 brand/logo/favicon.svg     -resize 32x32   public/images/favicon-32x32.png
magick -background none -density 600 brand/social/og-card.svg   -resize 1200x630 public/images/og-card.png
```

The shipped SVG `favicon.svg` already loads natively in all modern browsers; the PNG fallbacks above are only needed for legacy Safari versions, the PWA install splash, the email logo, and the Open Graph card.
